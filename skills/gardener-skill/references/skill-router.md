# Skill 路由决策树

## 核心判断

| 评估维度 | 操作流程型 Skill | 思维方式型 Skill |
|---|---|---|
| 结构 | 有「第X步」线性流程 | 无步骤，是节点+提问 |
| 内容 | 有输入/输出/检查点 | 有反面案例、边界说明 |
| 评估 | 可量化打分 | 不可量化，靠人类判断 |
| 优化工具 | 达尔文（分数棘轮） | 园丁（偏好判断） |

## 快速判断

**满足以下任一 → 思维方式型 → 用园丁**：
- Frontmatter/正文有「保护模式」「不需要达尔文式优化」声明
- 正文无「第一步→第二步」结构，有 NODE 标注
- 包含「如果...呢」「想象」「启发者」等关键词

**不满足任一 → 操作流程型 → 用达尔文**：
- 有明确的第一步→第二步线性流程
- 有输入/输出/检查点模板
- 可量化「输出质量是否符合宣称」

## 重要发现（2026-06-01）

- **达尔文的保护模式对 Anna 无效**：Anna 正文仍有「第一步→第二步→第三步」结构，保护模式识别时产生分歧。解决方式：彻底移除五步架子，改为纯节点结构。
- **思维方式型 Skill 不适合达尔文棘轮**：核心价值（启发性、创作空间）不可量化，棘轮机制无法运作。
- **解决路径**：建立独立的「园丁」工具，用人类偏好判断替代打分棘轮。
