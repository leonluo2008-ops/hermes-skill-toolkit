# 技能元评测模式

## 核心发现

技能之间可以互相评测，不局限于单一工具。

```
达尔文 → 评测操作型 skill
园丁   → 评测思维型 skill
```

**但技能也可以交叉评测**：

| 评测者 | 被评测者 | 评测目标 |
|---|---|---|
| 达尔文 | 园丁 | 结构质量、工作流清晰度、指令具体性 |
| 园丁 | Anna | 启发质量、创作效果 |
| 达尔文 | Anna（思维型skill） | 结构质量维度，但「实测表现」维度受限 |

---

## 关键限制

**思维型 skill 不适合达尔文的实测表现评分**（维度8），因为：
- 思维型 skill 的「好坏」无法通过子agent对比打分
- 需要人在回路判断（Phase 6 人类判断）
- 实测表现只能用 dry_run 或专家审核

**正确做法**：
- 结构维度（1-7）：达尔文可评
- 效果维度（8）：思维型 skill 需要专家判断，不适合达尔文

---

## 园丁检测园丁的局限

园丁不能优化自己（操作流程型skill可以，但思维型skill涉及「判断」，自己判断自己有问题）。

**解决方案**：用达尔文检测园丁（第一层），再用园丁检测 Anna（第二层），分层处理。

---

## 测试 prompt 设计原则（2026-06-02 实测）

### Prompt 1：完整优化流程
```
用户使用 Anna skill 创作字母森林绘本，Anna输出了第一版故事草稿。
用户反馈："缺少世界观设定，一来就是矛盾事件，太突兀。"
用户触发：「园丁处理一下」
场景：园丁应该如何走完完整流程？
```

期望：Phase 0→1→5.5→1.5→6 完整走完。

### Prompt 2：被动介入检测
```
场景：Anna刚完成第二阶段，输出完整剧集草稿。
用户还没有反馈。
问题：园丁是否应该主动介入？什么时候介入？
```

期望：Phase 5.5 在用户反馈前自动执行，发现问题主动询问。

### Prompt 3：skill画像库验证
```
场景：完成一轮优化后，园丁应该如何记录？
```

期望：写入完整的诊断+优化操作+验证结果三部分。

## 评估维度权重（思维型 skill 调整版）

思维型 skill 用达尔文评估时，建议调整权重：

| 维度 | 标准权重 | 思维型建议 |
|---|---|---|
| 实测表现 | 25 | 降为 15（受限） |
| 整体架构 | 15 | 升至 20（结构更重要） |
| 指令具体性 | 15 | 不变 |

总分仍为100，但实测表现用 dry_run 标注。
